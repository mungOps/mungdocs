    Docker #



    Docker 기본 명령어 #


    컨테이너 생성 # 

    $ docker run [OPTION] NAME[:TAG|@DIGEST]
    OPTION 특징
    -d, –detach 백그라운드에서 실행합니다.
    -i, –interactive 상호적으로 실행하며, 표준 입력을 유지합니다.
    -t, –tty 할당된 tty를 사용하여 실행합니다.
    –name 컨테이너에 이름을 할당합니다.
    -e, –env 컨테이너 내에서 사용할 환경 변수를 설정합니다.
    –rm 컨테이너가 종료되면 자동으로 삭제합니다.
    –network 컨테이너가 사용할 네트워크를 지정합니다.
    –publish, -p 호스트와 컨테이너 간의 포트 매핑을 설정합니다.
    –volume, -v 호스트와 컨테이너 간의 볼륨 매핑을 설정합니다.
    –help 도움말을 표시합니다.


    Network 생성 # 

    $ docker network create [OPTIONS] NETWORK
    옵션 설명
    -d 네트워크 드라이버 지정 (기본값: bridge)
    –subnet 네트워크의 서브넷 설정
    –gateway 네트워크의 게이트웨이 IP 설정
    –ip-range 네트워크의 IP 범위 설정
    –attachable 기존 네트워크에 컨테이너 연결 허용
    –internal 외부에서의 네트워크 접근 제한
    –label 네트워크에 메타데이터 설정
    –opt 드라이버별 옵션 설정
    –ipv6 IPv6 지원 활성화
    –ipam-driver IP 주소 관리 드라이버 지정
    –ipam-opt IP 주소 관리 옵션 설정
    –attachable 기존 네트워크에 컨테이너 연결 허용
    –internal 외부에서의 네트워크 접근 제한
    –ingress 인그레스 네트워크 생성
    –ingress-opt 인그레스 옵션 설정


    volume 생성 # 

    $ docker volume create [OPTIONS] STORAGE
    옵션 설명
    –storage-driver 사용할 스토리지 드라이버 설정
    –volume-driver 사용할 볼륨 드라이버 설정
    –add-registry 이미지를 가져올 도커 레지스트리 추가
    –disable-content-trust 이미지에 대한 콘텐츠 신뢰 기능 비활성화
    –icc 컨테이너 간 통신 허용 여부 설정
    –live-restore 도커 데몬 재시작 시 컨테이너 자동 복구 설정
    –log-driver 로깅 드라이버 설정
    –log-level 로깅 레벨 설정
    –storage-opt 스토리지 드라이버 옵션 설정
    –selinux-enabled SELinux 지원 여부 설정
    –storage-opt 스토리지 드라이버 옵션 설정
    –tls TLS 보안 연결 설정
    –tlscacert TLS CA 인증서 파일 경로 설정
    –tlscert TLS 인증서 파일 경로 설정
    –tlskey TLS 키 파일 경로 설정
    –tlsverify TLS 연결 검증 설정


    컨테니어 시작 # 

    $ docker start [OPTION]
    OPTION 특징
    -d, –detach 백그라운드에서 실행합니다.
    -i, –interactive 상호적으로 실행하며, 표준 입력을 유지합니다.
    -t, –tty 할당된 tty를 사용하여 실행합니다.
    –name 컨테이너에 이름을 할당합니다.
    -e, –env 컨테이너 내에서 사용할 환경 변수를 설정합니다.
    –rm 컨테이너가 종료되면 자동으로 삭제합니다.
    –network 컨테이너가 사용할 네트워크를 지정합니다.
    –publish, -p 호스트와 컨테이너 간의 포트 매핑을 설정합니다.
    –volume, -v 호스트와 컨테이너 간의 볼륨 매핑을 설정합니다.
    –help 도움말을 표시합니다.


    컨테이너 목록 확인 # 

    $ docker ps [OPTION]
    OPTION 특징
    -a, –all 모든 컨테이너를 출력합니다. (종료된 컨테이너 포함)
    -q, –quiet 컨테이너 ID만 출력합니다.
    –no-trunc 출력을 자르지 않고 전체 정보를 표시합니다.
    -n, –last n 최근 n개의 컨테이너만 표시합니다.
    -s, –size 각 컨테이너의 크기 정보를 표시합니다.
    –format 사용자 지정 포맷으로 출력합니다.
    –filter 필터를 적용하여 특정 조건을 만족하는 컨테이너만 표시합니다.
    –help 도움말을 표시합니다.


    컨테이너 중지 # 

    $ docker stop [OPTIONS] CONTAINER [CONTAINER...]
    OPTION 특징
    -t, –time 정지 프로세스에 대한 타임아웃(초)을 설정합니다. 기본값은 10초입니다.
    –help 도움말을 표시합니다.


    컨테이너 제거 # 

    $ docker rm [OPTIONS] CONTAINER [CONTAINER...]
    OPTION 특징
    -f, –force 실행 중인 컨테이너를 강제로 중지하고 제거합니다.
    -v, –volumes 컨테이너와 관련된 볼륨을 함께 제거합니다.
    –link 컨테이너를 다른 컨테이너에 연결한 경우 해당 연결을 제거합니다.
    –help 도움말을 표시합니다.


    이미지 확인 # 

    $ docker images [OPTIONS] [REPOSITORY[:TAG]]
    OPTION 특징
    -a, –all 모든 이미지를 표시합니다.
    –digests 각 이미지의 다이제스트 정보를 표시합니다.
    –format 사용자 지정 포맷으로 출력합니다.
    –no-trunc 출력을 자르지 않고 전체 정보를 표시합니다.
    -q, –quiet 이미지 ID만 출력합니다.
    –help 도움말을 표시합니다.


    이미지 다운로드 # 

    $ docker pull [OPTIONS] NAME[:TAG|@DIGEST]
    OPTION 특징
    -a, –all-tags 이미지의 모든 태그를 가져옵니다.
    –disable-content-trust 이미지의 내용 신뢰 기능을 비활성화합니다.
    –help 도움말을 표시합니다.


    이미지 삭제 # 

    $ docker rmi [OPTIONS] IMAGE [IMAGE...]


    컨테이너 접속 # 

    $ docker exec [OPTIONS] CONTAINER COMMAND [ARG...]
    OPTION 특징
    -d, –detach 백그라운드에서 실행합니다.
    -i, –interactive 상호적으로 실행하며, 표준 입력을 유지합니다.
    –env 컨테이너 내에서 사용할 환경 변수를 설정합니다.
    –user 명령을 실행할 사용자 또는 UID를 지정합니다.
    –workdir 명령을 실행할 작업 디렉토리를 지정합니다.
    –tty 할당된 tty를 사용하여 실행합니다.
    –help 도움말을 표시합니다.


    컨테이너 로그 확인 # 

    $ docker logs [OPTIONS] CONTAINER
    OPTION 특징
    –details 로그 항목의 추가 세부 정보를 표시합니다.
    -f, –follow 실시간으로 로그를 출력하며, 계속해서 갱신됩니다.
    –since 특정 시간 이후의 로그만 표시합니다.
    –tail 지정된 개수의 최근 로그 항목만 표시합니다.
    -t, –timestamps 로그 항목에 타임스탬프를 표시합니다.
    –until 특정 시간 이전의 로그만 표시합니다.
    –help 도움말을 표시합니다.


    이미지 생성 # 

    $ docker commit [OPTION] [CONTAINER] [NAME]
    OPTION 특징
    -a, –author 커미트한 사용자를 지정합니다.
    -c, –change 이미지 생성 시 Dockerfile 형식으로 변경을 추가합니다.
    -m, –message 커밋에 대한 설명 또는 메시지를 추가합니다.
    -p, –pause 커밋 시 컨테이너를 일시 중지합니다.
